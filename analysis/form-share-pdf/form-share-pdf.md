### Primary purpose

Field users need to share a filled form with a subject (patient, beneficiary) or a third party (e.g. a lab sharing a test result with a patient). Currently there is no way to export or share an individual form fill from the Avni Android app.

---

### Current state

The subject profile page lists all form fills for a subject (observations, encounters, etc.). Each row shows the form type and date. There is no share or export action available on any form fill.

---

### The feature

A share action is available on every form fill in the subject profile's form list. Tapping the share icon generates a PDF of that form fill in a table layout and invokes the Android native share sheet.

**Acceptance Criteria**

- A share icon is visible on each form fill row in the subject profile's form fills list
- Tapping the share icon generates a PDF of that form fill in portrait orientation
- The PDF opens with a subject header showing the subject's name and demographic line (gender, age, address hierarchy) — the same data visible above the action icons on the subject profile page
- Below the subject header, the form name and visit metadata (date, filled by, visit type) is shown
- The PDF uses a table layout: one row per field, with the field label in the left column and the filled value in the right column; fields are grouped under their section headers
- The PDF includes a watermark at the bottom: `Generated by Avni · <logged-in user name> · <date and time>` (e.g. `Generated by Avni · Raju Kumar · 20-04-2025 14:32`)
- The native Android share sheet is invoked with the PDF attached, allowing the user to share via WhatsApp, Gmail, or any installed app
- Every form type is shareable — no configuration required
- The share icon is available regardless of whether the form fill is in draft or completed state

---

### Out of scope

- iOS support
- Printing (physical printer output)
- Sharing blank/empty form templates
- Sharing from DEA
- Bulk sharing (multiple form fills at once)
- Custom layout per form type — all forms use the same table layout

---

### Technical details

**PDF generation**

Same approach as custom card share — use `react-native-html-to-pdf` or equivalent. The form fill data (field labels + values) is assembled into an HTML table at share time and converted to PDF.

**Table layout structure**

```
| Field label        | Value              |
|--------------------|---------------------|
| Survey date        | 05-12-2025          |
| Name               | Raju Kumar          |
| Age                | 34                  |
| Test result        | Positive            |
| Notes              | Follow up in 2 weeks|
```

**PDF header structure**

1. **Subject block** (dark background): subject name + demographic line (gender · age · address hierarchy) — pulled from the subject profile header, same data shown above the action icons on the profile page
2. **Form block** (light background): form type name + visit date, filled by, visit type
3. **Form sections**: section headers with field/value tables below each

```
┌──────────────────────────────────────┐
│  Adam D                              │  ← subject name
│  Male · 33 Years · Beed, Anjanwati  │  ← demographics + address
├──────────────────────────────────────┤
│  Farmer Endline                      │  ← form type
│  Visit Date: 04 Feb 2026 · Unplanned│
├──────────────────────────────────────┤
│  DETAILS                             │  ← section header
│  NGO Name      │ ...                 │
│  Farm area     │ ...                 │
├──────────────────────────────────────┤
│  FARMER DETAILS                      │
│  Village       │ ...                 │
│  ...                                 │
└──────────────────────────────────────┘
  Generated by Avni · User · datetime    ← watermark
```

**PDF filename**

Named after the form type + date: e.g. `lab_result_05_12_2025.pdf`

**Watermark**

`Generated by Avni · <user name> · <date time>` injected as a footer before PDF conversion.

**Portrait orientation**

PDF generated in portrait mode. Two-column table layout fits naturally in portrait.

**Share icon placement**

Share icon on each form fill row in the subject profile list. Tapping generates the PDF inline and immediately invokes the share sheet — no intermediate preview screen.

**Affected repositories**

| Repository | Change |
|---|---|
| `avni-client` | Share icon on form fill rows; HTML table assembly from form fill data; PDF generation; Android share intent |

### Open questions

1. **Section headers:** If a form has sections, should section names appear as separator rows in the table?
2. **Media fields:** Photo/audio/video fields are **omitted** from the PDF — they are not shown at all in the generated output.
3. **Multi-select fields:** How should multi-select values be displayed in a single table cell — comma-separated, or one per line?
4. **Field ordering:** Should fields appear in the same order as in the form, or grouped by section?
5. **Skip logic:** If a field was hidden due to skip logic and not filled, should it appear in the PDF or be omitted?
